    Introduction

    Short introduction to Lab 1

    What we want to achieve in this lab...

    Switch to Modeling Perspective

    1. After a fresh installation of Eclipse you will get the Welcome Page
    3. Next switch to the Modeling Perspective
    4. Click on the Menu "Window" on top of Eclipse
    6. Click on Open Perspective then on Other
    8. Dialog opens with list of perspectives
    10. Scroll down and Select "Modeling" from the list
    12. The Modeling Perspetive opens. You can also switch Perspectives using the icon on the top right corner

    Create a project

    1. Click on Create a new Model Project or Create Project
    4. In the case you selected "Create a Project" a wizard will be shown.
    5. Search and select the modeling project type
    8. Provide a name for your new project.
    10. Select "General" from Categegories and "Blank Package" from Templates
    12. Have a look at your freshly created new Project

    Basic UML Models and Basic UML Diagrams

    1. If Project is collapsed, click on the > symbols to uncollapse Project and Elements
    5. Add new Class to Model by right clicking the Package and select the class entry in the UML menu
    8. This creates a fresh new class. Name this new class "Customer"
    11. Next add a new attribute to this class using the context menue of the element and name it "custID"
    15. Now create an additional attribute and name it "custName"
    17. Create a new Diagram to manage all customer related model artifacts and name it "CustomerStore_Class"
    21. Add our "Customer" class to this diagram by using the context menue of the item
    24. Don't forget to save all your changes!

    How to import existing artifacts

    1. Right click Project to open context menu and select Import
    3. In this lab we want to import from the filesystem, please select "file system" in the wizard dialog
    5. click on Next
    6. In the file system dialog browse for the "Simple-Model" folder from your repository
    8. click on Browse and select "Simple-Model"
    10. click on Open
    11. The Import dialog opens
    13. Select the folder on the left (checkbox) and the model element will be automatically selected.
    15. Click on Finish to start the import
    16. The modeling artifact is imported and "SimpleUMLModel" is shown.
    18. Doublelick on the newly created Model and a migration dialog will open as the artifacts were created with older version.
    20. Just accept the default settings and the diagram will be opened
    22. To have a better view, right click on the canvas to open context menue ans select "Arrange All"

    How to add a new UML artifact to existing diagram

    1. Adding new UML artifacts to a diagram can be done by using the "Palette" or the context menue of the diagram.
    2. First we will use the "Palette"
    3. On the right side of DevOps Model Architect you will find the "Palette" menue.
    4. Click on "Class" entry to get all UML class types available
    5. Drag and drop the "Class" element from the Palette to the open diagram.
    6. A fresh new class is added and the name is selected for change. Change the name to "APPF2"
    7. Now we want to create a relationship from "APPF2" to "AppFunction1"
      1. click and drag the "incomming" icon to "AppFunction1"
    8. A new dialog opens and use "Create Instance" type
    9. The relation "create instance" is established between these two elements
    10. Do not forget to save your changes!

    Conclusion of Lab 1

    In this Lab we have been introduced how to use DevOps Model Architect to create a project, work on simple diagrams and import existing artifacts.
